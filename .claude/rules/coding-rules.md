# 编写规则

## 核心原则

- 严格遵守用户指定的任务需求，避免添加未经请求的功能
- 不要过度设计：不添加边界条件、验证逻辑、错误处理（除非明确需要）
- 默认使用最简单直接的解决方案
- 不创建不必要的文件、类、函数或配置

## 功能解决方案设计

**编写代码前**，以纯文本形式概述 1-3 个最佳实践解决方案：

- 每个方案用 3-5 句话描述：核心概念、优点、缺点、适用场景
- **等待用户确认或选择方案后**才生成代码

## TypeScript 编码风格

### 语法版本
- 使用最新稳定版 TypeScript 5.x

### 命名规范
- 变量/函数：`camelCase`
- 类/接口：`PascalCase`

### 代码风格（Airbnb）
- 4 个空格缩进
- 单引号
- 使用 `const` 声明，重新赋值时用 `let`，避免 `var`
- 异步操作使用 `async/await` 而不是 `.then`

## 注释规范

- 为所有状态变量和函数添加 JSDoc 注释，描述业务功能
- 简单代码（getter/setter、单行代码）省略注释
- 使用 `/** xxx */` 进行简要注释
- 复杂函数使用结构化 JSDoc
- 业务逻辑代码中不编写示例实现或单元测试（除非明确要求）
- 不使用 `console.log` 或 `debug` 语句（除非必要）

## 样式规则

### CSS 框架
- 所有样式使用 **Tailwind CSS** 或 **UnoCSS**（根据项目配置）
- 禁止使用原生 CSS（内联样式、.css 文件）
- 禁止使用其他框架（Less、Sass）

### HTML 标签规范
为所有有意义的标签添加 `data-alt` 属性：

```html
<div data-alt="main-content" class="p-4 bg-gray-100">...</div>
<button data-alt="submit-button" class="btn-primary">提交</button>
```

- `data-alt` 值应简短、准确、具有描述性
- 非语义标签（空包装器）可忽略
- 代码分析时指出缺失的 `data-alt` 属性
