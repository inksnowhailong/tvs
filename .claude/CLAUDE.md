## 项目核心架构

**设计理念**：业务与 UI 解耦，一套 TypeScript 业务逻辑服务所有端。

### 目录结构

项目划分为三个核心区域：

- **core**：业务逻辑层
  - 存放跨平台的核心业务逻辑
  - 包含请求封装、状态管理、工具函数等
  - 平台无关，专注业务本身

- **views**：视图层
  - 存放各端的 UI 实现
  - 仅处理 UI 交互和页面状态
  - 所有业务逻辑调用 core 层

- **utils**：工具库
  - 存放通用工具函数和类型定义
  - 可被 core 和 views 共用
  - 此处的工具必须不涉及业务逻辑，纯粹为开发提供便利
